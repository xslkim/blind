## 1. 游戏概述

### 1.1 核心信息

| 项目 | 内容 |
|------|------|
| 游戏名称 | 盲侠（Blade in Darkness） |
| 类型 | 音频动作格斗 |
| 平台 | iOS / Android |
| 技术栈 | Unreal Engine 5 + Steam Audio |
| 目标受众 | 动作游戏玩家、格斗游戏爱好者、视障玩家 |
| 游戏时长 | 主线 5-7 小时 |
| 定价模式 | 买断制 $8.99 |

### 1.2 Elevator Pitch

> 双目虽盲，心眼已开。用耳朵代替眼睛，听风声辨敌意，听呼吸知距离，听兵器破空判生死。《盲侠》是一款完全依赖听觉的动作格斗游戏——看不见的，才最真实。

### 1.3 核心体验目标

- **听声辨位的战斗快感**：每一次格挡、闪避、反击都完全基于声音信息，达成击杀时获得极强的成就感
- **节奏化的战斗**：Boss 战更像是"音乐游戏"，每个 Boss 有独特的攻击节奏模式
- **武者的成长**：从被杂兵欺负到击败最终 Boss，玩家自身（而非角色属性）在成长

## 2. 核心玩法

### 2.1 战斗循环

```
静听（判断敌方位、距离、动作）→ 反应（格挡/闪避/攻击）→ 命中反馈 → 连招 → 终结
```

### 2.2 核心机制

#### 2.2.1 听觉战斗系统

**敌情判断的三层信息**：

| 信息层 | 声源 | 可判断内容 | 响应时间窗口 |
|--------|------|-----------|------------|
| 位置层 | 呼吸声、脚步声 | 方位、距离 | 持续可用 |
| 动作预告层 | 衣物摩擦、兵器破风 | 攻击方向、攻击类型 | 攻击前 0.3-0.5 秒 |
| 状态层 | 呼吸急促度、心率 | 血量、体力、是否虚弱 | 持续可用 |

**攻击方向与音频特征映射**：

| 攻击方向 | 预告音效 | 破风声频率 | 玩家格挡方向 |
|---------|---------|-----------|------------|
| 上段斩（头顶） | 衣物上拉摩擦声 | 高频啸叫（下降） | 双指上滑 |
| 中段刺（正胸） | 脚步前冲声 | 中频呼啸 | 双指前滑（默认） |
| 下段扫（双腿） | 地面沙石摩擦声 | 低频嗡声 | 双指下滑 |
| 横斩（左右） | 身体扭转衣物声 | 高频从左/右划过 | 双指左/右滑 |

#### 2.2.2 体力系统

- **体力值范围**：0-100，不通过 HUD 显示
- **音频反馈**：
  - 体力 > 50%：呼吸声正常
  - 体力 20-50%：呼吸声变重，间隔缩短
  - 体力 10-20%：可听到明显的喘息声
  - 体力 < 10%：心跳声介入，操作响应有 0.1 秒延迟
- **消耗规则**：
  - 轻攻击：-5
  - 重攻击：-15
  - 闪避：-20
  - 格挡：-10（成功格挡返还 5）
  - 凝神状态：每秒 -8
  - 体力自动恢复：每秒 +3（非战斗状态 +8）

#### 2.2.3 内力系统

- **内力值范围**：0-3（气槽格数）
- **获取方式**：
  - 完美格挡（时机误差 < 0.1 秒）：+1
  - 连击 ≥ 5 段：+1
  - 闪避致命攻击：+1
- **消耗方式**：
  - 绝招：-3（全屏音效，对所有敌人造成高伤害）
  - 凝神状态激活：-1
  - 心眼反击（极限闪避后）：-1
- **音频反馈**：内力满时听到低频嗡鸣（"内力充盈"的感觉）

#### 2.2.4 凝神状态（核心特色机制）

激活后效果（持续 5 秒）：
- 时间流速降低至 40%（所有声音播放速度变慢）
- 环境噪声降低 80%
- 敌人的呼吸声和动作预告音放大 50%
- 允许玩家在敌人攻击动画中途识别并做出反应（正常速度下几乎不可能）

**代价**：消耗 1 格内力，且冷却时间 30 秒

## 3. 触控交互设计

### 3.1 战斗手势表

| 手势 | 功能 | 判定窗口 | 触觉反馈 |
|------|------|---------|---------|
| 单指 ↑ 滑 | 上段斩 | - | 轻振 |
| 单指 ↓ 滑 | 下段扫 | - | 轻振 |
| 单指 ← 滑 | 左横斩 | - | 轻振 |
| 单指 → 滑 | 右横斩 | - | 轻振 |
| 单指 ↑ 快速重滑 | 上段重斩 | - | 重振 |
| 单指点击 | 中段刺（快速突刺） | - | 短振 |
| 双指 ↑ 滑 | 上段格挡 | 攻击前 0.5s | 双振 |
| 双指 ↓ 滑 | 下段格挡 | 攻击前 0.5s | 双振 |
| 双指 ← 滑 | 左格挡 | 攻击前 0.5s | 双振 |
| 双指 → 滑 | 右格挡 | 攻击前 0.5s | 双振 |
| 双击 | 后撤闪避 | - | 双短振 |
| 长按（> 1s） | 激活凝神状态 | - | 持续微振 → 进入后单振确认 |

### 3.2 手势识别优化

- 滑动手势识别阈值：最小滑动距离 20px，最大识别时间 300ms
- 轻/重攻击判定：滑动速度 > 500px/s = 重击，否则 = 轻击
- 格挡与攻击区分：双指触控优先识别为格挡，单指为攻击
- 误触防护：双指操作时单个手指短暂抬起不取消格挡判定

### 3.3 非战斗手势

| 手势 | 功能 |
|------|------|
| 三指点击 | 打开角色状态（内力/体力语音播报） |
| 双指长按 | 坐下冥想（快速恢复体力） |
| 单指在屏幕画圈 | 360° 转动角色（听周围环境） |
| 摇晃手机 | 甩刀回鞘（战斗结束动作，触发评价语音） |

## 4. 音频设计

### 4.1 空间音频规范

- **引擎**：Steam Audio（支持 HRTF 双耳渲染 + 物理声学）
- **HRTF 预设**：默认 KEMAR，提供 3 种可选的 HRTF 配置文件
- **声源优先级**：
  - P0（最高）：Boss 攻击预告音
  - P1：近身敌人攻击音
  - P2：玩家自身动作音
  - P3：远处敌人位置音
  - P4：环境音

### 4.2 武器音效组

#### 玩家武器（5 种可选）

| 武器 | 破风声 | 命中音 | 格挡音 | 特性 |
|------|--------|--------|--------|------|
| 长剑 | 高频哨音 | 利刃入肉声 | 清脆金属撞击 | 均衡，攻击快 |
| 重刀 | 低频呼啸 | 沉重劈砍声 | 沉闷撞击 | 伤害高，速度慢 |
| 双匕 | 极轻咻声 | 细密切割声 | 几乎无声（难以格挡） | 速度快，伤害低 |
| 长棍 | 中频嗡声 | 钝击声 | 空心木击声 | 范围大，可击退 |
| 链刃 | 锁链声 + 破风声 | 撕裂声 | 锁链缠绕声 | 中距离，连击强 |

#### 敌人武器（12 种）

每种敌人武器有独特的预告音效，玩家需要在战斗中"听"出对方持什么兵器。

### 4.3 动态战斗音乐

- **分层系统**：战斗音乐由 5 个音乐层组成
  - Layer 1（基底）：打击乐循环——始终播放
  - Layer 2（节奏层）：在敌人攻击前 0.5 秒加入预示节奏
  - Layer 3（旋律层）：在玩家连击时渐入
  - Layer 4（高潮层）：在凝神状态 / 内力全满时进入
  - Layer 5（终结层）：Boss 残血时进入
- **节奏同步**：Boss 战的背景音乐节奏与 Boss 的攻击节奏对齐
  - 例："铁爪"Boss 的攻击节奏是 3/4 拍（慢-慢-快），背景打击乐就是 3/4 拍
  - 玩家可以通过数拍子预判攻击时机

### 4.4 语音内容规划

| 内容 | 数量 | 说明 |
|------|------|------|
| Boss 战台词 | 每个 Boss 30-50 句 | 含战斗挑衅、受伤、愤怒、濒死、死亡 |
| 剧情对话 | ~8000 字 | 7 章，每章 1-2 个叙事场景 |
| 杂兵语音 | 通用组 20 句 × 3 种声线 | 发现玩家、攻击怒吼、受伤、死亡 |
| 旁白 | ~2000 字 | 关键剧情转折的叙述 |
| 系统语音 | ~100 条 | 教程、状态播报、成就提示 |

## 5. 叙事与世界观

### 5.1 世界观

> 架空的武侠世界。三十年前，"天目诀"——一种以听觉替代视觉的武功心法——被江湖视为邪术，传人遭灭门。你是唯一的幸存者，被一位隐世盲侠救下并抚养长大。如今师父已逝，你下山寻找灭门真凶。

### 5.2 七章结构

| 章节 | 环境 | Boss | 核心机制引入 |
|------|------|------|------------|
| 一·下山 | 山林 | 野狼群（合围战） | 基础移动、攻击、格挡 |
| 二·渡口 | 码头 | "铁爪"——原朝廷鹰犬 | 面对武器种类差异 |
| 三·雨夜 | 城镇 | "暗镖"——暗器高手 | 雨声干扰下的战斗 |
| 四·古刹 | 寺庙 | "铜钟"——僧人 | 凝神状态引导 |
| 五·冰窟 | 雪山 | "雪女"——轻功高手 | 极寒环境（呼吸成冰的脚步声） |
| 六·深渊边 | 悬崖 | "无名"——师父的旧识 | 内力系统完整使用 |
| 七·真相 | 灭门遗址 | "师叔"——最终 Boss | 所有机制融合 |

### 5.3 分支结局

| 结局 | 条件 | 最终选择 |
|------|------|---------|
| 侠道 | 全章节不杀平民 NPC | 放过师叔，让他说出真相后自尽 |
| 修罗 | 杀了师叔 | 独自离开，但被仇恨吞噬 |
| 传承 | 收集全部师父的录音遗言 | 重建师门，收徒传天目诀 |
| 解脱 | 在最终战前选择放下剑 | 离开江湖，远走他乡 |

## 6. 关卡设计

### 6.1 战斗场景类型

| 场景类型 | 声学特征 | 战术影响 |
|---------|---------|---------|
| 开阔地 | 无回声，声音衰减快 | 远程武器优势 |
| 密闭室 | 强回声，混响长 | 近战优势，回声干扰定位 |
| 水边 | 水声底噪持续 | 低音被掩蔽，格挡音不易听到 |
| 风中 | 风噪间歇 | 高频信息被掩蔽，攻击预告音减弱 |
| 石地 | 脚步声清脆 | 移动信息完全暴露 |
| 草地 | 脚步声低 | 可隐匿移动 |
| 雪地 | 脚步声沉闷，有积雪压实的特殊声 | 所有人移动声放大 |

### 6.2 遭遇战设计规则

- 杂兵战：2-5 个敌人，用于练习当前章节的核心机制
- 精英战：1 个强化敌人 + 1-2 个杂兵
- Boss 战：单一 Boss，多阶段（3-4 阶段），每阶段剩余 25% 血量时切换
- 每章包含：3-4 场杂兵战 + 1 场精英战 + Boss 战

## 7. 技术规格

| 参数 | 规格 |
|------|------|
| 引擎 | Unreal Engine 5 |
| 空间音频 | Steam Audio |
| 音频延迟目标 | < 30ms（战斗反馈） |
| 触觉反馈 | iOS Core Haptics / Android VibrationEffect |
| 最低系统 | iOS 15 / Android 11（Steam Audio 性能要求较高） |
| 安装包 | < 2GB |
| 帧率目标 | 不适用（无渲染压力），但音频线程需稳定 60fps tick |

## 8. 无障碍

- 完整触觉战斗教程（约 10 分钟）
- 三种难度模式，最低难度下 Boss 攻击前有语音提示（"上段攻击来了！"）
- 支持自定义 HRTF 校准（通过简单的声音定位测试匹配玩家耳朵）
- 完全兼容系统屏幕朗读
