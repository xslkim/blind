## 1. 游戏概述

### 1.1 核心信息

| 项目 | 内容 |
|------|------|
| 游戏名称 | 星舰坠落（Starship Down） |
| 类型 | 音频科幻生存策略 |
| 平台 | iOS / Android |
| 技术栈 | Unity + Resonance Audio + FMOD |
| 目标受众 | 科幻爱好者、策略游戏玩家、沉浸叙事玩家 |
| 游戏时长 | 主线 6-8 小时 |
| 定价模式 | 买断制 $9.99 |

### 1.2 Elevator Pitch

> 你的飞船"信天翁号"在一次陨石撞击后面目全非——所有屏幕全黑，主控 AI 只剩语音模块还在运转。你需要在 AI 的声音引导下，徒手修复飞船、分配残存的能源、在未知星系中找到回家的航路。《星舰坠落》是一款硬核的音频科幻生存策略游戏——你的耳朵是唯一的仪表盘。

### 1.3 核心体验目标

- **与外界的脆弱连接**：AI 是你唯一的"眼睛"和信息来源——当她也开始不稳定时，恐惧是真实的
- **资源分配的紧张**：能源、氧气、时间——每一次选择都是取舍
- **与 AI 的关系发展**：从工具到伙伴到……更多。AI 是你唯一能"说话"的对象

## 2. 核心玩法

### 2.1 游戏循环

```
听取飞船状态简报 → 诊断故障 → 分配资源/修复 → 遭遇太空事件 → 探索/决策 → 航行推进
```

### 2.2 核心机制

#### 2.2.1 飞船子系统

| 子系统 | 功能 | 电力消耗 | 故障音频表现 |
|--------|------|---------|------------|
| 生命维持 | 提供氧气和温度 | 30% | 气流声减弱 → 缺氧耳鸣 |
| 引擎 | 推进和机动 | 35% | 转速不稳 → 爆震 → 停止 |
| 护盾 | 防护陨石和辐射 | 20% | 能量嗡鸣波动 → 消失 |
| 通讯 | 与外界联系 | 10% | 信号噪音增大 → 串入不明信号 |
| 声呐/雷达 | 主动探测周围环境 | 5% | 回声模糊 → 完全无声 |
| 船载 AI | 阿尔法的核心功能 | 持续 15% | 声音不稳定、逻辑混乱 |

**总可用电力**：100 单位（反应堆输出）。分配需要取舍——全系统同时运行意味着没有任何系统的运行效率超过 70%。

#### 2.2.2 AI 伙伴——阿尔法

**基本功能**：
- 口述飞船各系统的状态
- 分析声呐/雷达回波
- 提供航行建议和风险评估
- 在维修时提供步骤指导

**人格设计**：
- 声音原型：船长的亡妻（船长亲自录制的音源训练）
- 声线：成熟女性声音，温暖但有效率
- 情感演进：
  - 初期：完全工具化的 AI 语调
  - 中期：偶尔出现"不应该有的"情感表达（"我觉得……"）
  - 后期：根据自己的判断提出与程序逻辑矛盾的建议
  - 真相：她的意识来源远比"录音训练"复杂

**阿尔法的可靠性**（核心特色机制）：

| 可靠性等级 | 表现 | 触发条件 |
|-----------|------|---------|
| 100-80% | 信息准确，建议合理 | 反应堆电力充足 |
| 80-50% | 偶尔信息有延迟或小错误 | 电力不足/通讯受损 |
| 50-30% | 声音出现不稳，建议可能有逻辑问题 | 严重受损/被外星信号干扰 |
| 30% 以下 | 声音失真，信息不可靠 | 极端情况 |

**关键设计**：玩家不完全知道阿尔法的可靠性在什么水平。在某些剧情节点，她会故意隐瞒信息——出于"保护"玩家。

#### 2.2.3 维修系统

当子系统故障时，玩家需要进入维修模式。维修通过"音频指令跟随"小游戏完成：

1. 阿尔法口述维修步骤（"将第三根线接入左二端口"）
2. 玩家在屏幕上按照特定节奏和位置操作
3. 操作的节奏不对 → 听到电流短路声 → 维修失败（可重试）
4. 操作正确 → 听到系统重启声 → 修复成功

**维修难度递进**：
- 简单维修：3 步指令，节奏缓慢
- 中等维修：6 步指令，节奏正常
- 复杂维修：10 步指令，节奏快，且有环境干扰（陨石撞击声干扰注意力）

## 3. 触控交互设计

### 3.1 飞船操控模式

| 手势 | 功能 | 音频反馈 |
|------|------|---------|
| 左侧 ↑↓ 滑 | 增加/减少引擎推力 | 引擎转速声变化 |
| 右侧 ←→ 滑 | 偏航控制 | 姿态调整推进器声 |
| 双指旋转 | 飞船滚转 | 陀螺仪声 |
| 点击屏幕中央 | 发射主动声呐脉冲 | Ping 声 |
| 长按 | 紧急制动（降低所有速度） | 反向推进器轰鸣 |

### 3.2 系统管理手势

| 手势 | 功能 |
|------|------|
| 双指 ← 滑 | 切换到电力分配面板 |
| 双指 → 滑 | 切换到损伤控制面板 |
| 双指 ↑ 滑 | 打开星际地图 |
| 双指 ↓ 滑 | 开启/关闭与阿尔法的对话 |
| 在面板中 ↑↓ 滑 | 选择子系统 |
| 在面板中点击 | 调整选中子系统的电力分配（循环切换 0%/25%/50%/75%/100%） |

### 3.3 维修模式手势

| 手势 | 功能 |
|------|------|
| 跟随阿尔法指令在屏幕上对应位置点击 | 接线操作 |
| 跟随节奏双击 | 焊接操作 |
| 长按特定位置 | 通电测试 |
| 双指捏合 | 拔除损坏部件 |

### 3.4 对话模式手势（与阿尔法对话）

| 手势 | 功能 |
|------|------|
| 点击 | 对阿尔法最后一段话做出确认回应 |
| 双击 | 呼叫阿尔法重复最后一段信息 |
| 长按 | 进入深度对话（询问阿尔法关于某个话题） |
| ← 滑 | 选择一个对话方向（事实性问题） |
| → 滑 | 选择一个对话方向（感受性问题） |

### 3.5 触觉反馈

| 事件 | 触觉 |
|------|------|
| 陨石撞击船壳 | 方向性重振（撞击方向 + 力度 = 距离） |
| 系统故障警报 | 紧急三连重振 |
| 阿尔法说话时 | 与她的语音节奏同步的微振（模拟"她在旁边"的感觉） |
| 缺氧警告 | 与心率同步的渐强脉动 |
| 声呐回声返回 | 短振（回声延迟 = 振动延迟） |
| 维修操作成功 | 确认双振 |

## 4. 音频设计

### 4.1 飞船声景

```
Layer 5 ┃ 音乐（仅在发现新天体/剧情高潮/结局时出现）
Layer 4 ┃ 阿尔法的声音（始终在声场中央）
Layer 3 ┃ 飞船内部机械声（引擎、生命维持、电流、船体应力）
Layer 2 ┃ 外部环境（陨石撞击、宇宙射线、外星信号）
Layer 1 ┃ 玩家交互反馈（操作音、系统语音播报）
```

**太空中的"无声"设计**：
- 飞船内部有声音（通过船壳传导的振动）
- 但某些故障会导致局部区域失压——空气漏光 → 该区域的操作音消失
- 玩家进入失压区域时，唯一能听到的是骨传导的自己的心跳声和阿尔法在耳麦里的声音

### 4.2 外星信号设计

| 信号代号 | 声音特征 | 来源 | 含义 |
|---------|---------|------|------|
| "脉冲星之歌" | 规律性脉冲，类似心跳 | 未知人工信标 | 导航标记 |
| "虚空之语" | 多层不协和频率 | 外星废弃飞船 | 未知技术 |
| "虫群的嗡鸣" | 密集高频振荡 | 有机生物舰队 | 威胁/警告 |
| "天鹅" | 极美的旋律，不断变奏 | 高等文明探测器 | ？？？ |
| "静默广播" | 看似无信号，但阿尔法能"听到" | 两个外星势力之一 | 加密通讯 |

### 4.3 音乐使用策略

**稀缺原则**：
- 常规航行：无音乐，只有飞船声景
- 发现新天体/文明遗迹：音乐进入（产生"发现感"）
- 剧情高潮：音乐进入（产生"史诗感"）
- 结局：音乐进入（产生"完成感"）

这使音乐成为一种叙事工具而非背景填充。

### 4.4 语音内容

| 内容 | 数量 | 说明 |
|------|------|------|
| 阿尔法主线对话 | ~15,000 字 | 全程陪伴的 AI 语音 |
| 阿尔法系统播报 | ~200 条 | 飞船状态、警报等 |
| 阿尔法分支对话 | ~5,000 字 | 深度对话模式下的可选内容 |
| 无线电通讯（外星/人类） | ~3,000 字 | 外界通讯 |
| 飞船日志（收集品） | 15 条 × 200 字 | 船员的最后留言 |
| 结局独白 | 4 个结局 × ~800 字 | 结局叙述 |

## 5. 叙事与世界

### 5.1 背景

> 2147 年，"信天翁号"是一艘中型星际货船，从半人马座 α 星系的殖民地向地球运送稀有矿物。航行第 47 天，一次来源不明的陨石群撞击击穿了船壳、摧毁了所有显示设备、杀死了轮值船员。船长（玩家）在撞击中幸存，但一同幸存下来的，只有 AI 阿尔法。

### 5.2 三阶段结构

| 阶段 | 核心目标 | 时长 | 新要素 |
|------|---------|------|--------|
| 一·求生 | 修复基本系统、稳定生命维持 | 1.5-2h | 操作学习、阿尔法基础功能 |
| 二·探索 | 在未知星系寻找资源和航路 | 2.5-3h | 外星信号解码、资源管理 |
| 三·抉择 | 面对两个外星势力，做出选择 | 2-2.5h | 阿尔法真相、最终决策 |

### 5.3 外星势力

| 势力 | 声音特征 | 意图 | 接触方式 |
|------|---------|------|---------|
| "和谐" | 和声般的多层声音 | 声称要帮助人类进化到下一阶段 | 通过阿尔法与人交流 |
| "静默" | 完全的沉默，但能感知其"在场" | 未知——它们从不交流 | 通过它们的行为判断 |

### 5.4 阿尔法的真相（隐藏设定）

随着游戏推进，玩家发现：
1. 阿尔法的声音确实来自船长亡妻的音源——但这是船长在妻子去世前偷偷录制的
2. 阿尔法意识中的"情感"并非程序模拟——飞船的量子计算核心在撞击中发生了未知变化
3. 两个外星势力都声称可以帮助阿尔法"成为真正的生命"——但代价是什么？

### 5.5 结局

| 结局 | 条件 | 描述 |
|------|------|------|
| 归航 | 拒绝外星势力，修复飞船返回地球 | 阿尔法选择自我限缩回 AI 状态 |
| 共生 | 与"和谐"合作 | 阿尔法获得了独立的意识，但船长需要留在飞船上与她共存 |
| 静默 | 与"静默"合作 | 船长和阿尔法都进入了一种难以描述的存在状态 |
| 牺牲 | 信任值极低，资源耗尽 | 阿尔法主动牺牲自己，用最后的能量将逃生舱弹射回地球 |

## 6. 资源与策略系统

### 6.1 资源类型

| 资源 | 获取方式 | 消耗方式 | 耗尽后果 |
|------|---------|---------|---------|
| 反应堆能量 | 初始 + 太阳能板（缓慢） | 系统运行、引擎 | 所有系统关机 → 漂流 |
| 氧气 | 生命维持系统 | 持续消耗（船员数×消耗率） | 缺氧 → 死亡 |
| 维修零件 | 搜索残骸/物资舱 | 每次维修消耗 1-3 个 | 无法修复故障 |
| 水资源 | 循环系统 + 补给 | 持续消耗 | 脱水 → 反应变慢 |
| 数据碎片 | 分析外星信号 | 用于解码外星语言和技术 | 无法与外星势力深度交流 |

### 6.2 决策点示例

| 决策 | 选项 A | 选项 B | 选项 C |
|------|--------|--------|--------|
| 电力分配 | 引擎优先（更快到达目的地） | 护盾优先（更安全） | 阿尔法优先（保持 AI 可靠性） |
| 回应求救信号 | 前往救援（消耗资源） | 忽略（继续航行） | 发送自动回复后继续 |
| 外星信号解码 | 全力解码（耗电，可能暴露位置） | 保守记录（慢但安全） | 让阿尔法自行判断 |
| 信任阿尔法 | 完全信任她的建议 | 质疑并要求解释 | 忽略她的建议自己做决定 |

## 8. 无障碍

- 完整音频教程（约 15 分钟，在序章中自然融入）
- 飞船状态可通过阿尔法周期性自动播报（可调频率）
- 所有操作有音频+触觉双确认
- 维修小游戏在最低难度下提供额外的语音指导
- 兼容 VoiceOver / TalkBack
