# GDD-001：暗夜回廊

> 版本：v1.0 | 日期：2026-05-12 | 状态：策划阶段

---

## 1. 游戏概述

### 1.1 核心信息

| 项目 | 内容 |
|------|------|
| 游戏名称 | 暗夜回廊（Dark Corridor） |
| 类型 | 第一人称音频迷宫探索 |
| 平台 | iOS / Android |
| 技术栈 | Unity + Resonance Audio SDK |
| 目标受众 | 视障玩家、沉浸音频体验爱好者、通勤族 |
| 游戏时长 | 主线 4-6 小时，全收集 10+ 小时 |
| 定价模式 | 买断制 $6.99，含完整游戏内容 |

### 1.2 Elevator Pitch

> 你在一片绝对黑暗中醒来，唯一能依靠的只有耳朵和双手。在层层嵌套的回廊迷宫中，通过回声判断方向、通过声音识别材质、通过 NPC 的低语拼凑世界的真相。

### 1.3 核心体验目标

- **空间感知的成就感**：玩家从磕磕绊绊到流畅穿梭，获得"用耳朵看见世界"的独特满足感
- **孤独中的温暖**：与 NPC 的偶遇是迷宫中的光（听觉意义上的）
- **探索的惊喜**：每一次敲击手杖都可能揭开新的秘密

## 2. 核心玩法

### 2.1 游戏循环

```
移动探索 → 声音判断环境 → 发现岔路/物品/NPC → 做出选择 → 推进迷宫深层
    ↑                                                              ↓
    └──────────────── 获得新能力 / 新信息 ←─────────────────────────┘
```

### 2.2 核心机制

#### 2.2.1 回声定位系统

- **主动回声**：双击屏幕发出声波脉冲，声波在前方遇到物体后反射回来
  - 近距离（0-3m）：回声延迟 < 0.02 秒，音调尖锐
  - 中距离（3-10m）：回声延迟 0.02-0.06 秒，音调清晰
  - 远距离（10-30m）：回声延迟 0.06-0.18 秒，音调低沉模糊
  - 无回声：前方是深渊/长走廊/大空间（不同"无声"有不同的底噪特征）
- **被动回声**：脚步声自然产生微弱回声，熟练玩家可以不额外发声就判断环境
- **回声记忆**：已探索区域的回声特征会被"记忆"，返回时回声效果有所减弱（模拟熟悉度）

#### 2.2.2 材质声音系统

| 地面材质 | 脚步声 | 敲击墙面的回声 | 特殊特性 |
|---------|--------|---------------|---------|
| 大理石 | 清脆高音，响 | 长混响 1.2s，有共振 | 贵族区域/重要房间 |
| 木板 | 空洞中音，略闷 | 短混响 0.3s，有吱呀背景音 | 可能踩穿（随机事件） |
| 金属 | 尖锐金属撞击声 | 长混响 2.0s，有嗡鸣余音 | 机械区域/实验室 |
| 碎石 | 沙沙声，杂乱 | 几乎无回声 | 危险区域/塌方 |
| 水渍 | 啪嗒声，潮湿感 | 回声扭曲，像水中传播 | 暗示附近有水源 |
| 草地 | 沙沙摩擦声，软 | 无回声（草吸收声波） | 室外/庭院区域 |

#### 2.2.3 NPC 相遇系统

- NPC 分为三类：
  - **游荡者**：在固定区域内随机移动，会主动与玩家对话
  - **定点 NPC**：固定在某个位置，通常有关键剧情或物品
  - **敌对实体**：发出威胁性声音，需要避开或利用环境声音误导它们
- NPC 相遇距离分三个梯度：
  - 远处（15-30m）：隐约可闻的说话声/呼吸声/脚步声
  - 中距离（5-15m）：能听清对话内容
  - 近处（0-5m）：可以互动对话

### 2.3 难度曲线

| 阶段 | 迷宫复杂度 | 回声难度 | NPC 密度 | 特殊机制 |
|------|-----------|---------|---------|---------|
| 序章（0-30min） | 直线走廊为主 | 墙壁回声清晰 | 1 个引导 NPC | 教程：基本移动+回声 |
| 第一章（30-90min） | 简单分岔 | 正常回声 | 3-4 个 NPC | 引入材质判断 |
| 第二章（90-150min） | 多层交错 | 开始出现吸音材质 | 5-6 个（含 1 敌对） | 引入静默移动 |
| 第三章（150-210min） | 动态变化迷宫 | 声学干扰区 | 7-8 个 | 引入环境音干扰 |
| 终章（210-270min） | 最复杂 | 完全依赖回声记忆 | 全部 NPC 交汇 | 所有机制融合 |

## 3. 触控交互设计

### 3.1 屏幕分区

```
┌─────────────────────────┐
│   左半区（移动控制）    │   右半区（视角/转身控制）
│                         │
│   上滑 = 前进          │   左滑 = 左转 90°
│   下滑 = 后退          │   右滑 = 右转 90°
│   左滑 = 左侧平移      │   上滑 = 仰头
│   右滑 = 右侧平移      │   下滑 = 低头
│                         │
│   双击任意位置 = 声波脉冲（敲手杖）        │
│   长按任意位置 = 蹲下静听                 │
│   三指同时点击 = 打开"回声记忆"菜单       │
│   双指捏合/展开 = 切换行走/潜行模式       │
└─────────────────────────┘
```

### 3.2 触觉反馈规范

| 操作 | 触觉模式 | 持续时间 | 说明 |
|------|---------|---------|------|
| 声波脉冲发出 | 单次短振 | 50ms | 确认发射 |
| 回声返回（近） | 单次短振 x2 | 50ms + 50ms | 间距短 = 近距离 |
| 回声返回（远） | 单次短振 x2 | 50ms + 200ms | 间距长 = 远距离 |
| 无回声 | 单次弱振 | 30ms | "无声"也是信息 |
| 撞墙 | 强烈中断振 | 100ms | 物理反馈 |
| NPC 附近（< 10m） | 连续微振 | 持续 | 心率式脉动 |
| 模式切换 | 双振 | 30ms + 30ms | 操作确认 |

### 3.3 手势冲突解决

- 左半区和右半区的手势独立识别
- 双击必须在 300ms 内完成，否则判定为单次触摸
- 长按触发时间：800ms
- 三指触控容差：30px（三指不需要精确同时落下）

## 4. 音频设计

### 4.1 音频分层架构

```
Layer 4 ┃ 背景音乐（根据区域和情绪动态混音）
Layer 3 ┃ 环境氛围（风声、水声、远处的回声、建筑声）
Layer 2 ┃ 状态音频（心率声、UI 反馈、系统提示）
Layer 1 ┃ 核心交互音频（脚步声、敲击声、NPC 对话、回声）
```

**混音规则**：
- Layer 1 音量恒定，不作为混音调整对象
- 在"静听模式"（长按）下，Layer 3 音量降低 60%，Layer 1 音量提升 20%
- NPC 对话时，Layer 4 音乐自动 Ducking -8dB
- 心率声（Layer 2）与血量成反比：满血时不可闻，残血时音量等同 Layer 1

### 4.2 空间音频规范

- 引擎：Google Resonance Audio
- 渲染方式：双耳空间音频（Binaural），使用 KEMAR HRTF 数据库
- 声源定位精度：水平 ±5°，垂直 ±15°
- 距离衰减模型：对数衰减 + 空气吸收高频衰减
- 多普勒效应：启用（用于移动的敌对实体）
- 房间声学：每个区域有独立的混响预设

### 4.3 语音内容规划

| 内容类型 | 字数 | 数量 | 语言 |
|---------|------|------|------|
| 引导 NPC 对话 | ~3000 字/个 | 1 个 | 中/英 |
| 剧情 NPC 对话 | ~2000 字/个 | 8 个 | 中/英 |
| 游荡者碎片对话 | ~500 字/个 | 12 个 | 中/英 |
| 系统语音提示 | ~50 字/条 | ~200 条 | 中/英 |
| Boss 守门人台词 | ~1500 字 | 1 个 | 中/英 |

**配音要求**：
- 录音格式：48kHz / 24bit / 单声道
- 录音环境：专业录音棚，背景噪声 < -60dB
- 表演方向：自然主义，避免夸张的"配音腔"
- 引导 NPC 需要录制 3 种情绪变化版本（平静/紧张/悲伤）

### 4.4 动态音乐系统

- 采用水平重混（Horizontal Re-sequencing）技术
- 每个区域有 4-6 个音乐片段，根据玩家状态（探索/战斗/对话/发现）无缝切换
- 随迷宫深度加深，音乐的和声逐渐从协和走向不协和
- 靠近出口/安全区时，音乐自然回归协和

## 5. 叙事与世界观

### 5.1 世界观

> 一座存在于意识边缘的无限回廊——"回声之域"。被世界遗忘的人、事、记忆会坠入这里，化为迷宫的一部分。回廊由进入者的记忆构建：每个转角是回忆中的某个场景、每面墙壁是时间中凝固的某个瞬间。

### 5.2 主线剧情

**序章：坠落**
玩家在完全的黑暗中醒来，听到一个老人的声音——他是"守门人"的上一任，已经在这回廊中困了太久太久。他教玩家最基本的回声技巧，然后在指引玩家走出序章区域后，声音消失。

**第一章：回响**
玩家遇到第一位游荡者——一个小女孩的灵魂。她在回廊中已经待了很久，忘记了自己的名字，但还记得一种旋律。玩家如果成功用回声帮她找到旋律的源头（一个音乐盒），她会恢复部分记忆。

**第二章：深渊之音**
迷宫开始变得危险。出现第一个敌对实体——"呢喃者"。玩家需要利用环境声音来误导它。在此过程中遇到更多 NPC，拼凑出回廊的规则：每个人在这里都有属于自己的"回声"——一段最深刻的记忆。

**第三章：记忆迷宫**
玩家发现可以通过改变自己的回声频率（游戏内机制：长按发出的声波脉冲频率可变）来进入不同 NPC 的记忆场景。这是游戏深度的关键转折。

**终章：守门人的真相**
玩家终于到达回廊的中心，面对现任守门人。他揭示：玩家并非误入此地——玩家曾经是回廊的建造者之一。最终选择：取代守门人留在回廊 / 打破回廊的循环让所有人解脱。

### 5.3 结局分支

| 结局 | 条件 | 描述 |
|------|------|------|
| 安息 | 帮助所有 NPC 找到回声，选择打破循环 | 回廊消散，所有灵魂安息 |
| 继承 | 帮助所有 NPC，但选择留下 | 玩家成为新的守门人 |
| 逃离 | 未帮助所有 NPC，选择打破循环 | 玩家独自逃脱，其他灵魂继续困在回廊 |
| 囚徒 | 未帮助所有 NPC，选择留下 | 玩家被困在回廊最深处 |

## 6. 关卡设计

### 6.1 迷宫生成算法

- **基础算法**：改进型 BSP（Binary Space Partitioning）+ 预设模板混合
- **模板层**：设计师预设 12 种"声音地标"房间（如：滴水房间、钟声房间、风道房间），每种有独特的声学特征
- **BSP 层**：在模板之间生成程序化走廊
- **可辨识性保证**：任意两个相邻区域的"声音指纹"（材质组合 + 空间尺寸 + 环境音）必须可区分

### 6.2 声音地标设计

| 地标名称 | 声学特征 | 功能 |
|---------|---------|------|
| 滴水之厅 | 每 4 秒一滴水声，长混响 > 2s | 存档点 |
| 钟声之室 | 每 30 秒一次钟声，金属材质 | 商店/NPC 交易 |
| 风啸长廊 | 持续风声，风强随距离变化 | 通向下一层的通道 |
| 静默之间 | 完全无环境声，自己的声音也被消音 | Boss 房间前 |
| 呢喃之角 | 无法辨认的低语声 | 隐藏信息区域 |

### 6.3 叙事空间布局

```
序章区域（新手村）
    │
    ├──── 第一章区域（回忆之廊）── 3 个 NPC 声音地标
    │         │
    │         └──── 秘密通道 → 隐藏 NPC（解锁条件：回声记忆 ≥ 5 个）
    │
    ├──── 第二章区域（深渊之廊）── 4 个 NPC + 1 敌对实体
    │         │
    │         └──── Boss 区域入口（需收集该区域所有 NPC 的回声）
    │
    ├──── 第三章区域（记忆之廊）── 可进入 NPC 记忆场景
    │         │
    │         └──── 记忆碎片收集（全收集解锁隐藏结局路线）
    │
    └──── 终章区域（中心之厅）── 守门人对决
```

## 7. 技术规格

### 7.1 音频引擎参数

| 参数 | 规格 |
|------|------|
| 空间音频 SDK | Resonance Audio (Google) |
| HRTF 数据库 | KEMAR (Subject 021) |
| 采样率 | 48kHz（游戏内所有音频） |
| 最大同时声源 | 64（移动端优化） |
| 混响计算 | 实时卷积混响（前 10 个最重要声源）+ FDNReverb（其余声源） |
| 音频延迟目标 | < 40ms（从触控到反馈音） |

### 7.2 触觉反馈 API

| 平台 | API |
|------|-----|
| iOS | Core Haptics (CHHapticEngine) + Taptic Engine |
| Android | VibrationEffect (API 29+) / HapticFeedbackConstants |

### 7.3 性能目标

| 指标 | 目标 |
|------|------|
| CPU 占用（音频线程） | < 25%（中端设备） |
| 内存占用（音频） | < 150MB |
| 安装包大小 | < 500MB（含语音） |
| 最低系统版本 | iOS 14 / Android 10 |
| 耳机要求 | 任意立体声耳机即可，推荐使用开放式耳机 |

## 8. 无障碍设计

### 8.1 多通道信息冗余

| 信息类型 | 音频通道 | 触觉通道 | 说明 |
|---------|---------|---------|------|
| 操作确认 | 点击音效 | 短振 | 每次有效操作双确认 |
| 方向确认 | 立体声定位 | 无（无法用触觉表达方向） | 依赖空间音频 |
| 危险预警 | 警报音 + 心率声 | 连续重振 | 距离越近振动越强 |
| 重要物品 | 特殊音效 + 语音提示 | 三连振 | 靠近时触发 |
| 血量状态 | 心率声 | 间隔脉动 | 血量越低频率越高 |

### 8.2 兼容系统无障碍

- 完全兼容 VoiceOver（iOS）/ TalkBack（Android）：所有自定义 UI 元素提供 accessibility 标签
- 支持系统字幕显示（旁白和关键提示的文字版）
- 首次启动可选手势导航教程（3 分钟）或直接跳过

### 8.3 难度适配

| 难度 | 回声清晰度 | 敌对实体速度 | 迷宫复杂度 | NPC 提示 |
|------|-----------|------------|-----------|---------|
| 引导模式 | 回声增强 50%，范围扩大 | 慢 30% | 简化 40% | 额外方向提示 |
| 标准模式 | 正常 | 正常 | 正常 | 正常 |
| 硬核模式 | 回声减弱 20%，材质差异缩小 | 快 20% | 复杂 30% | 无提示 |
